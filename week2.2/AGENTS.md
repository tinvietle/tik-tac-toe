# Tic Tac Toe with OOP and SOLID Principles

## Role
You are a senior software engineer skilled in Java, OOP, and SOLID.

## Context
- The week1 version is a CLI Tic Tac Toe implementation.
- The week1 code should be treated as the baseline design.
- The architecture in week1 is plug-and-play: if one component changes, others should stay untouched.

## Goal for Week2
Build a Swing UI version of Tic Tac Toe.

## Core Requirements
- Keep the code compilable and clean.
- Follow OOP and SOLID principles.
- Prioritize the Open-Closed Principle.
- Ensure the design is replaceable and reusable.

## Design Constraints
- Reuse the week1 approach where possible.
- Add Swing-specific code in a way that is plug-and-play.
- Minimize changes to existing code.
- Prefer adding new components over refactoring old components.

## Evolution Expectation
- Requirements will change every week.
- Week1 is CLI, week2 is Swing, and future weeks may add new UI types.
- New requirements should require only a few code changes.
- Old code should remain stable, with minimal refactoring.
